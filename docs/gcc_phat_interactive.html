Kéo các thanh trượt để thay đổi tín hiệu nguồn và độ trễ thật, rồi xem từng bước công thức biến đổi như thế nào. Khớp 1-1 với GCC_PHAT() trong main.c.

2FFT → phổ X_a, X_b

X_a = FFT(x_a) , X_b = FFT(x_b) // arm_rfft_fast_f32(...,0)

Trễ trong miền thời gian ⇔ một độ dốc pha tuyến tính trong miền tần số: X_b(f) = X_a(f)·e^(−j·2π·f·D). Biên độ |X| gần như giữ nguyên; thông tin trễ nằm hết ở pha.

3Cross-spectrum: R = conj(X_a)·X_b

conj(X_a)·X_b = (ar − j·ai)(br + j·bi) re = ar·br + ai·bi im = ar·bi − ai·br

Nhân liên hợp X_a với X_b. Pha của R(f) = pha(X_b) − pha(X_a) = −2π·f·D — chính là độ dốc chứa độ trễ. Đây đúng 2 dòng re/im trong code.

pha của R(f) (độ dốc ∝ −D)

4PHAT whitening: chia cho biên độ

R_phat(f) = R(f) / |R(f)| (|R| ← max(|R|, 1e-9) chống chia 0)

Bỏ hết biên độ, chỉ giữ pha (mỗi bin thành véc-tơ đơn vị). Mọi tần số được "cân bằng" như nhau → đỉnh tương quan nhọn và bền với phổ nguồn / vang. Đổi PHAT sang TẮT để thấy đỉnh tù đi.

|R| trước|R_phat| sau (=1 khi BẬT)

5IFFT → hàm tương quan r[n]

r[n] = IFFT( R_phat ) // arm_rfft_fast_f32(...,1) lag = argmax_n r[n] // arm_max_f32 ; n > N/2 ⇒ lag âm

Biến đổi ngược về miền thời gian. Vị trí đỉnh = độ trễ D. Trục đã được cuộn (wrap) để delay âm nằm bên trái 0.
